1. INTRODUCTION
   ============================================================

## 1.1 Purpose

The purpose of this document is to define the complete functional
and non-functional requirements of the SkillBridge application
with detailed user stories, validations, and acceptance criteria.

This SRS serves as:

* A development reference document
* A testing and validation baseline
* An interview explanation artifact
* A blueprint for enterprise-style system design

---

## 1.2 Scope

SkillBridge is a web-based full-stack application that allows users
to track job applications, interview rounds, outcomes, and notes.

The system provides:

* Secure user authentication and authorization
* Job application lifecycle management
* Interview round tracking
* Dashboard analytics
* CI/CD integration and code quality enforcement

3. FUNCTIONAL REQUIREMENTS (USER STORIES)
=========================================

---

## US-01: User Registration (Sign-Up)

Description:
As a new user, I want to register using my email and password so that
I can securely access the platform.

Input Fields:

* Email
* Password
* Confirm Password

Validations:

* Email must not be null or empty
* Email must follow valid format
* Email must be unique
* Password must be at least 8 characters
* Password must include uppercase, lowercase, digit, special character
* Confirm password must match password

Acceptance Criteria:

1. User submits registration form
2. System validates all inputs
3. On validation failure, HTTP 400 returned
4. If email already exists, HTTP 409 returned
5. Password encrypted using BCrypt
6. User saved with USER role
7. HTTP 201 Created returned

---

## US-02: User Login (Sign-In)

Description:
As a registered user, I want to log in to receive a JWT token.

Validations:

* Email and password must not be empty
* Email must exist
* Password must match encrypted password

Acceptance Criteria:

1. User submits login credentials
2. System validates credentials
3. Invalid credentials return HTTP 401
4. Valid login generates JWT
5. Token includes userId, role, expiry
6. User redirected to dashboard

---

## US-03: Secure API Access (JWT Authorization)

Description:
As a user, I want all protected APIs to require authentication.

Acceptance Criteria:

* Missing token returns HTTP 401
* Invalid or expired token returns HTTP 401
* Insufficient role returns HTTP 403
* Valid token allows access

---

## US-04: Create Job Application

Description:
As a user, I want to add a job application.

Validations:

* Company name mandatory
* Job role mandatory
* Status must be valid enum
* Applied date cannot be future

Acceptance Criteria:

1. Authenticated user submits application
2. Application linked to logged-in user
3. Created timestamp saved
4. HTTP 201 Created returned

---

## US-05: View Job Applications

Description:
As a user, I want to view all my job applications.

Acceptance Criteria:

* Only user-owned applications returned
* Pagination supported
* Sorted by applied date

---

## US-06: Update Job Application

Description:
As a user, I want to update job application details.

Acceptance Criteria:

* Only owner can update
* Status must be valid enum
* Updated timestamp maintained

---

## US-07: Delete Job Application

Description:
As a user, I want to delete a job application.

Acceptance Criteria:

* Only owner can delete
* Related interviews deleted automatically
* HTTP 204 No Content returned

---

## US-08: Add Interview Round

Description:
As a user, I want to add interview rounds to an application.

Validations:

* Round type must be valid enum
* Interview date mandatory
* Interview date >= application date

Acceptance Criteria:

* Interview linked to application
* HTTP 201 Created returned

---

## US-09: Update Interview Round

Description:
As a user, I want to update interview notes and outcome.

Acceptance Criteria:

* Only owner can update
* Notes and result editable

---

## US-10: Dashboard View

Description:
As a user, I want to see a dashboard summary.

Acceptance Criteria:

* Total applications count
* Status-wise breakdown
* Offer and rejection counts

---

## US-11: CI Pipeline Execution

Description:
As a developer, I want automated builds using Jenkins.

Acceptance Criteria:

* Jenkins triggers on code push
* Maven build executed
* Unit tests run automatically

---

## US-12: Code Quality Analysis

Description:
As a developer, I want static code analysis via SonarQube.

Acceptance Criteria:

* SonarQube scan executed
* Quality gate enforced
* Build fails on violations
